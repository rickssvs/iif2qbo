1. Overview

The RO Writer IIF to QuickBooks Online Importer is a locally-run application that helps you import financial data from RO Writer 3 into QuickBooks Online. This Software is designed with privacy as a core principle:

  • No external servers: Your data never touches our servers
  • Local processing only: All file processing happens on your computer
  • Direct QuickBooks connection: Data goes directly from your computer to QuickBooks
  • No tracking or analytics: We do not track your usage or collect analytics

2. Information We Do NOT Collect

Rick's SVS Automotive LLC does NOT collect, store, or transmit:

  • Your financial data or IIF file contents
  • Your QuickBooks Online credentials (username/password)
  • Customer names, addresses, or personal information from your records
  • Transaction details, amounts, or account information
  • Usage analytics or behavioral data
  • IP addresses or device information
  • Cookies or tracking pixels

3. Information Stored Locally on Your Computer

The Software stores the following information in your web browser's localStorage (on your computer only):

Data Type Purpose Storage Location
QuickBooks OAuth Access Token Authenticate API requests to QuickBooks Online Browser localStorage
QuickBooks OAuth Refresh Token Renew access without re-authentication Browser localStorage
QuickBooks Company ID (Realm ID) Identify which QuickBooks company to access Browser localStorage
Token Expiration Time Know when to refresh the access token Browser localStorage
Environment Setting (Sandbox/Production) Remember which QuickBooks environment you're using Browser localStorage
Import Log (Filenames and Dates) Track which IIF files have been imported to prevent duplicates Browser localStorage
Account Mapping Cache Speed up imports by caching QuickBooks account IDs Browser localStorage
Important: All of this data is stored ONLY on your computer in your web browser. It is never transmitted to Rick's SVS Automotive LLC or any third party (except QuickBooks Online as needed for API authentication).

4. How Your Data is Processed

Here's exactly what happens when you use the Software:

4.1 OAuth Authentication

  1. You click "Connect to QuickBooks" in the application
  2. You are redirected to Intuit's official OAuth login page (appcenter.intuit.com)
  3. You log in with your QuickBooks credentials directly on Intuit's website
  4. Intuit redirects you back to the application with a temporary authorization code
  5. The application exchanges this code for access tokens via a local PowerShell server running on your computer (localhost:8080)
  6. The tokens are stored in your browser's localStorage

Your QuickBooks password never passes through our application.

4.2 IIF File Processing

  1. You select IIF files from your local file system
  2. The files are read directly in your browser using JavaScript
  3. The IIF data is parsed and converted to QuickBooks journal entry format
  4. The converted data is sent directly to QuickBooks Online via HTTPS
  5. No file data is stored permanently - it exists only in memory during processing

4.3 Data Flow Diagram

Your Computer → reads IIF files → parses locally → sends to → QuickBooks Online

Rick's SVS Automotive LLC is NOT in this data flow. We never see your data.

5.2 Local PowerShell Server

The Software uses a local PowerShell server (server.ps1) that runs on your computer at localhost:8080. This server:

  • Handles OAuth callbacks from Intuit
  • Proxies API requests to QuickBooks to avoid CORS issues
  • Runs entirely on your local machine
  • Does not log or store any data
  • Only accessible from your computer (localhost)

6. Data Security

We take the following measures to protect your data:

  • HTTPS Encryption: All communication with QuickBooks Online is encrypted via HTTPS
  • OAuth 2.0: Industry-standard authentication protocol (no passwords stored)
  • Local Processing: Your financial data never leaves your computer except to go directly to QuickBooks
  • No Cloud Storage: We do not use any cloud services to process or store your data
  • Token Security: OAuth tokens are stored in browser localStorage, which is isolated per-origin

Your Responsibilities:

  • Keep your computer and browser secure
  • Do not share your QuickBooks credentials with anyone
  • Log out of QuickBooks when finished
  • Use strong passwords for your QuickBooks account
  • Clear browser data if using a shared computer

7. Data Retention

Because we don't collect your data, we don't retain it. The only data stored is:

  • On your computer: OAuth tokens and import logs in browser localStorage
  • In QuickBooks: Journal entries you import (subject to Intuit's retention policies)

How to Delete Your Data:

To remove all locally stored data from the Software:

  1. Open the application in your browser
  2. Open browser Developer Tools (F12)
  3. Go to "Application" or "Storage" tab
  4. Find "Local Storage" → "http://localhost:8080"
  5. Delete all items or clear entire localStorage

Alternatively, clearing your browser's cache and data will remove all stored information.
